# Flask Aggregator

**Flask Aggregator** — лёгкий каталог для научных материалов (статей, диссертаций, изображений, аудио).  
Приложение умеет сканировать директорию, извлекать текст/транскрипты и даёт удобный интерфейс для просмотра коллекции с гибкими тегами и поиском.

## Возможности
- **Адаптивный интерфейс**: корректно работает на мобильных устройствах (сворачиваемая навигация, пагинация).
- **Гибкая система тегов**: произвольные теги в формате ключ/значение с автодополнением.
- **Предпросмотр**: встроенный просмотр текста, изображений, PDF и аудио + миниатюры.
- **Фасетный поиск**: фильтрация по типу, году и тегам; активные фильтры отображаются бейджами и очищаются одним кликом.
- **Тёмная/светлая тема**: переключение темы с сохранением предпочтения в браузере.
- **Фоновые операции**: тяжёлые задачи (сканирование, извлечение) выполняются в потоках, чтобы не блокировать UI.

## Технологии
- **Backend**: Python, Flask, SQLAlchemy, SQLite  
- **UI**: Bootstrap-стили  
- **Извлечение** (опционально): PyMuPDF (PDF/изображения), `pytesseract` (OCR), Whisper (аудио), и др.

> Опциональные извлекатели подключаются только при наличии соответствующих системных зависимостей.

## Требования
- Python 3.10+ (рекомендуется)
- Git (для клонирования — по желанию)
- Для некоторых возможностей:
  - **OCR**: установленный `tesseract-ocr`
  - **Аудио-транскрипция**: `ffmpeg` + модель Whisper

## Установка
```bash
python3 -m venv .venv
source .venv/bin/activate            # Windows (PowerShell): .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
cp .env.example .env                 # укажите SCAN_ROOT внутри
```

## Конфигурация
Откройте файл `.env` и укажите ключевые параметры:

- `SCAN_ROOT` — абсолютный путь к корневой папке с материалами, которые нужно индексировать/сканировать.

Пример:
```env
SCAN_ROOT=/home/user/datasets/library
```

## Инициализация БД и запуск
```bash
flask --app app.py init-db   # однократно для создания схемы БД
python app.py                # запустить сервер разработки
```

После запуска откройте в браузере адрес, указанный в консоли (обычно `http://127.0.0.1:5000/`).

> Для смены порта/хоста см. переменные окружения Flask (например, `FLASK_RUN_PORT=8000`) или используйте обёртки в README_DEPLOY.md.

## Использование
1. **Импорт или загрузка**: добавьте файлы через кнопки **Import** или **Upload** в интерфейсе.
2. **Сканирование**: запустите сканирование для извлечения текста/метаданных. Тяжёлые операции выполняются в фоновых потоках.
3. **Просмотр и поиск**: пролистывайте каталог, используйте поиск и фасетные фильтры. Пагинация защищает от загрузки слишком большого объёма данных за раз.
4. **Редактирование записей**: при необходимости исправляйте метаданные и управляйте тегами (добавляйте/удаляйте ключи и значения).
5. **Темы**: переключайте светлую/тёмную тему в интерфейсе; выбор сохраняется в браузере.

## Производительность
- **Фоновые потоки** используются для ресурсоёмких операций (OCR, извлечение из PDF/аудио).
- **Пагинация** по умолчанию ограничивает количество элементов на страницу.
- Рекомендуется держать SCAN_ROOT структурированным (подпапки по году/типу/источнику) — это упрощает навигацию и повторные сканы.
